# 工具页面设计文档

**日期：** 2026-04-23
**状态：** 设计阶段

---

## 概述

新增一个开发者工具页面，集合常用开发工具：JSON格式化/压缩/校验、时间戳转换、文本对比、URL编码/解码、Base64编码/解码。

**设计原则：**
- 极简风格，与现有博客页面保持一致
- 单页Tab切换式布局
- 混合处理模式（实时 + 按钮触发）
- 完整深色模式支持
- 移动端友好

---

## 1. 页面架构

### 文件位置
- **文件：** `src/pages/tools.astro`
- **路由：** `/tools/`

### 页面结构

```
┌─────────────────────────────────────┐
│  Header（继承自BaseLayout）          │
├─────────────────────────────────────┤
│  ┌─────────────────────────────────┐│
│  │  页面标题：🔧 开发者工具箱       ││
│  │  简介：常用开发工具集合          ││
│  └─────────────────────────────────┘│
│  ┌─────────────────────────────────┐│
│  │  Tab导航栏                      ││
│  │  [JSON] [时间戳] [Diff] [URL] [Base64]│
│  └─────────────────────────────────┘│
│  ┌─────────────────────────────────┐│
│  │  工具内容区（根据Tab动态切换）   ││
│  │                                 ││
│  │  输入区 + 输出区                ││
│  │                                 ││
│  └─────────────────────────────────┘│
```

### 技术选择
- 继承 `BaseLayout.astro` 保持风格统一
- Tab切换使用纯客户端JavaScript（无需框架）
- 所有状态存储在浏览器内存（无需localStorage）
- 使用Tailwind CSS样式，深色模式自动适配

## 2. Tab导航栏设计

### 视觉风格

```
┌──────────────────────────────────────────────────────┐
│ [ 📋 JSON ] [ 🕐 时间戳 ] [ 🔀 Diff ] [ 🔗 URL ] [ 📦 Base64 ] │
└──────────────────────────────────────────────────────┘
```

### 样式细节
- 圆角按钮风格，与todo页面的象限按钮一致
- 未选中态：`bg-gray-100 dark:bg-gray-800` 灰色背景
- 选中态：`bg-blue-600 dark:bg-blue-500` 蓝色背景 + 白色文字
- 每个Tab带emoji图标，增强辨识度
- 响应式：移动端Tab按钮缩小，保持flex-wrap
- 过渡动画：`transition-all duration-200` 平滑切换

### 交互行为
- 点击Tab切换工具，无页面刷新
- Tab切换时内容区淡入动画（opacity + transform）
- 支持键盘导航：← → 键切换Tab
- URL hash同步：切换Tab后URL变为 `/tools/#json`、`/tools/#timestamp` 等
- 页面加载时根据hash自动选中对应Tab

## 3. 工具一：JSON格式化/压缩/校验

### 基本信息
- **名称：** JSON工具
- **图标：** 📋
- **处理模式：** 按钮触发（格式化/压缩）+ 实时校验

### 布局

```
┌─────────────────────────────────────────────┐
│  ┌─────────────────────────────────────┐    │
│  │  输入区（textarea）                 │    │
│  │  placeholder: "请输入JSON字符串..." │    │
│  │  支持拖拽文件上传.json文件          │    │
│  └─────────────────────────────────────┘    │
│  ┌─────────────────────────────────────┐    │
│  │  操作按钮行                         │    │
│  │  [格式化] [压缩] [校验] [清空]      │    │
│  └─────────────────────────────────────┘    │
│  ┌─────────────────────────────────────┐    │
│  │  输出区（textarea/pre）             │    │
│  │  显示结果或错误信息                 │    │
│  └─────────────────────────────────────┘    │
│  ┌─────────────────────────────────────┐    │
│  │  [复制结果]                         │    │
│  └─────────────────────────────────────┘    │
└─────────────────────────────────────────────┘
```

### 处理逻辑
- **实时校验：** 输入时自动校验JSON有效性，有效时显示绿色✓图标
- **格式化：** 使用2空格缩进美化JSON
- **压缩：** 移除所有空格和换行，输出紧凑JSON
- **校验按钮：** 显式校验并高亮显示JSON错误位置

### 错误处理
- JSON无效时，输出区显示红色错误提示
- 错误信息包含：行号、列号、错误类型
- 使用 `try-catch` 捕获 `JSON.parse()` 错误

### 样式细节
- 输入/输出区使用等宽字体（monospace）
- 按钮：格式化（蓝色）、压缩（绿色）、校验（橙色）、清空（灰色）
- 深色模式：输入区 `bg-gray-900`

## 4. 工具二：时间戳转换

### 基本信息
- **名称：** 时间戳转换
- **图标：** 🕐
- **处理模式：** 实时处理

### 布局

```
┌─────────────────────────────────────────────┐
│  ┌─────────────────────────────────────┐    │
│  │  时间戳输入区                       │    │
│  │  输入框 + [毫秒/秒] 单位切换        │    │
│  │  placeholder: "请输入时间戳..."     │    │
│  └─────────────────────────────────────┘    │
│  ┌─────────────────────────────────────┐    │
│  │  实时转换结果区                     │    │
│  │  本地时间：2026-04-23 12:30:45      │    │
│  │  UTC时间：2026-04-23T04:30:45Z      │    │
│  │  相对时间：3天前 / 刚刚             │    │
│  │  ISO 8601：2026-04-23T12:30:45+08:00│    │
│  └─────────────────────────────────────┘    │
│  ┌─────────────────────────────────────┐    │
│  │  日期转时间戳区                     │    │
│  │  日期选择器 + 时间选择器            │    │
│  │  显示：对应的时间戳值               │    │
│  └─────────────────────────────────────┘    │
│  ┌─────────────────────────────────────┐    │
│  │  快捷按钮：[当前时间戳] [今天0点] [本周一]│
│  └─────────────────────────────────────┘    │
└─────────────────────────────────────────────┘
```

### 处理逻辑
- 支持**秒级**（10位）和**毫秒级**（13位）时间戳
- 单位切换：默认根据输入长度自动判断，可手动切换
- 实时转换：输入时实时显示转换结果（debounce 200ms）

### 转换结果展示
1. **本地时间：** `YYYY-MM-DD HH:mm:ss` 格式
2. **UTC时间：** ISO 8601 UTC格式
3. **相对时间：** "刚刚"、"3分钟前"、"2小时前"、"昨天"、"3天前"等
4. **ISO 8601：** 完整ISO格式带时区

### 反向转换
- 使用 `<input type="datetime-local">` 选择器
- 选择后实时显示对应的秒级/毫秒级时间戳

### 快捷按钮
- "当前时间戳"：生成当前时间戳并填入输入框
- "今天0点"：生成当天00:00:00的时间戳
- "本周一"：生成本周周一00:00:00的时间戳

## 5. 工具三：文本对比

### 基本信息
- **名称：** 文本对比
- **图标：** 🔀
- **处理模式：** 按钮触发

### 布局

```
┌─────────────────────────────────────────────┐
│  ┌──────────────┐  ┌──────────────┐         │
│  │  原文输入区   │  │  新文输入区   │         │
│  │  textarea    │  │  textarea    │         │
│  │  "原文..."   │  │  "新文..."   │         │
│  └──────────────┘  └──────────────┘         │
│  ┌─────────────────────────────────────┐    │
│  │  [开始对比] [清空全部]              │    │
│  └─────────────────────────────────────┘    │
│  ┌─────────────────────────────────────┐    │
│  │  对比结果统计                       │    │
│  │  新增：+5行  删除：-3行  修改：2行  │    │
│  └─────────────────────────────────────┘    │
│  ┌─────────────────────────────────────────┐│
│  │  对比结果展示区（左右分栏）             ││
│  │  ┌──────────────┐  ┌──────────────┐     ││
│  │  │原文（行号）  │  │新文（行号）  │     ││
│  │  │1: const a   │  │1: const a   │     ││
│  │  │2: [删除] let│  │2: [新增] var│     ││
│  │  │3: return    │  │3: return    │     ││
│  │  └──────────────┘  └──────────────┘     ││
│  └─────────────────────────────────────────┘│
└─────────────────────────────────────────────┘
```

### 对比算法（行级）
1. 将原文和新文按行分割
2. 逐行比较，标记：
   - **新增行（绿色背景）：** 新文中存在，原文中不存在
   - **删除行（红色背景）：** 原文中存在，新文中不存在
   - **未变化（无标记）：** 行内容相同

### 对比结果展示
- 左右分栏同步滚动
- 左栏显示原文带行号，右栏显示新文带行号
- 高亮行背景色：
  - 新增：`bg-green-100 dark:bg-green-900/30`
  - 删除：`bg-red-100 dark:bg-red-900/30`
- 行号区域固定宽度，内容区域自适应

### 统计信息
- 显示新增行数、删除行数、修改行数
- 使用彩色徽章：+绿色、-红色、~黄色

## 6. 工具四：URL编码/解码

### 基本信息
- **名称：** URL编码/解码
- **图标：** 🔗
- **处理模式：** 按钮触发

### 布局

```
┌─────────────────────────────────────────────┐
│  ┌─────────────────────────────────────┐    │
│  │  输入区                             │    │
│  │  placeholder: "请输入URL或字符串..." │    │
│  └─────────────────────────────────────┘    │
│  ┌─────────────────────────────────────┐    │
│  │  操作按钮行                         │    │
│  │  [编码] [解码] [清空]               │    │
│  └─────────────────────────────────────┘    │
│  ┌─────────────────────────────────────┐    │
│  │  输出区                             │    │
│  │  显示编码/解码结果                  │    │
│  └─────────────────────────────────────┘    │
│  ┌─────────────────────────────────────┐    │
│  │  [复制结果]                         │    │
│  └─────────────────────────────────────┘    │
└─────────────────────────────────────────────┘
```

### 处理逻辑
- **编码：** 使用 `encodeURIComponent()` 对输入字符串进行URL编码
- **解码：** 使用 `decodeURIComponent()` 对输入字符串进行URL解码
- 自动识别：如果输入包含 `%XX` 格式，提示"可能是已编码URL"

### 功能细节
- 支持完整URL编码（包含协议、路径、查询参数）
- 支持单个查询参数值编码
- 解码时自动处理嵌套编码（如 `%2520` → `%20` → 空格）

### 错误处理
- 解码失败时显示错误提示（如 `%E0` 无效UTF-8序列）
- 使用 `try-catch` 捕获解码错误

```
│  ┌─────────────────────────────────────┐    │
│  │  操作按钮行                         │    │
│  │  [编码] [解码] [清空]               │    │
│  └─────────────────────────────────────┘    │
│  ┌─────────────────────────────────────┐    │
│  │  输出区                             │    │
│  │  显示Base64编码/解码结果            │    │
```

### 处理逻辑
- **编码：** 使用 `btoa()` 或 `TextEncoder` + Base64编码

- 支持UTF-8字符编码（处理中文等非ASCII字符）

### 文件处理
- 文件上传：使用 `<input type="file">` 或拖拽
- 编码文件：显示Base64字符串
- 解码文件：显示文件大小信息 + [下载文件]按钮
- 大文件提示：超过1MB时警告用户

### 错误处理
- 解码失败时显示错误（如无效Base64字符）
- 文件过大时提示

## 8. 公共组件和样式

### 复制功能
- 所有工具共享统一的"复制结果"按钮
- 使用 `navigator.clipboard.writeText()` API
- 复制成功后显示Toast提示："已复制到剪贴板"（2秒后自动消失）

### Toast提示
- 固定在页面底部居中显示
- 样式：黑色背景 + 白色文字 + 圆角
- 动画：淡入淡出（0.3秒）

### 错误提示样式
- 红色边框 + 红色文字
- 包含错误图标（❌）
- 详细错误信息 + 解决建议

### 输入区样式
- 白色/深灰色背景
- 圆角边框（border-radius: 8px）
- focus时蓝色边框高亮
- placeholder灰色文字
- 等宽字体用于代码/数据展示

## 9. 响应式设计

### 移动端适配（< 768px）

**Tab导航栏：**
- Tab按钮缩小为紧凑样式
- 使用flex-wrap，Tab按钮可换行显示

**工具内容区：**
- JSON、时间戳、URL、Base64：输入区和输出区上下堆叠
- 文本对比：
  - 输入区上下堆叠
  - 对比结果区改为上下堆叠（原文在上，新文在下）
  - 保持同步滚动逻辑

**按钮布局：**
- 操作按钮行使用flex-wrap，按钮可换行

## 11. 性能优化

### 关键优化点
1. **延迟加载：** 工具内容区只在Tab切换时渲染
2. **防抖处理：** 时间戳工具的实时转换使用debounce（200ms）
3. **最小化DOM操作：** diff工具对比结果一次性插入
4. **避免内存泄漏：** Tab切换时清理事件监听器和定时器
5. **文件处理：** 大文件（>1MB）提示用户

---

## 12. 错误处理策略

### 统一错误处理
1. **输入验证：** 执行操作前验证输入有效性
2. **捕获异常：** 使用 `try-catch` 捕获所有操作异常
3. **友好提示：** 错误信息用用户友好语言描述
4. **恢复建议：** 提供错误恢复建议
5. **边界情况：** 处理空输入、超长输入、特殊字符等
